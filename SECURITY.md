# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| main    | ✅                  |

## Reporting a Vulnerability

If you discover a security vulnerability, please **do not open a public issue**. Instead:

1. Email the maintainers directly (add your contact email here), or
2. Use GitHub's [private vulnerability reporting](../../security/advisories/new) feature.

Please include:

- A description of the vulnerability
- Steps to reproduce
- Potential impact

We aim to respond within 5 business days and will keep you updated as the issue is investigated and resolved.

## Disclosure Policy

We ask that you give us a reasonable amount of time to address the issue before any public disclosure.
