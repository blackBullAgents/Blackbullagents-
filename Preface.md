# Preface

## About This Repository

BlackBull Agent documents the architecture behind an automated market-making system for crypto token teams. The goal of this repository is to give engineers and project teams a structured reference for how the system connects wallets, ingests fee revenue, and converts it into continuous liquidity support.

## How to Use This Documentation

The docs are organized into five parts, moving from onboarding concepts to operational detail:

1. **Part 1 — Getting Started**: connecting a wallet and selecting a token
2. **Part 2 — Fee Routing & Liquidity Engine**: how fees become liquidity
3. **Part 3 — Dashboard & Monitoring**: real-time market health tooling
4. **Part 4 — Security & Controls**: custody model, permissions, kill switch
5. **Part 5 — Case Studies & FAQ**: example deployments and common questions

Each chapter is self-contained, with diagrams and references back to the relevant section of the product where applicable.
